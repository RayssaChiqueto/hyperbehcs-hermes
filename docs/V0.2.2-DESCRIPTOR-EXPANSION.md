# v0.2.2 Descriptor Expansion

v0.2.2 expands public HyperBEHCS Hermes from the v0.2.1 doc/CI hardening seed into a broader descriptor surface.

This release still does not export local runtime or authority. It only adds public-safe descriptor packets and sidecars.

## Added descriptor packs

```text
packs/hermes/public_hermes_goal_descriptors.hbp
packs/browser/public_browser_descriptors.hbp
packs/endpoints/public_endpoint_descriptors.hbp
packs/proofs/public_proof_receipt_descriptors.hbp
```

## Public surfaces described

- Hermes `/Goal` intake, planning, delegation, and verification surfaces.
- Browser observation as separate from browser control, screenshot capture, and keyboard control.
- Endpoint metadata as separate from endpoint open, network call, and webhook open.
- Proof/receipt concepts for chain, promotion, revoke, expire, and cosign/quorum.

## Authority posture

Every new row keeps execution/control fields closed:

```text
dispatch=0
route=0
shell=0
terminal=0
file_write=0
memory_write=0
tool_execute=0
skill_execute=0
mcp_execute=0
webmcp_execute=0
provider_call=0
endpoint_open=0
browser_control=0
keyboard_control=0
screenshot_capture=0
network_call=0
webhook_open=0
cron_create=0
device_read=0
device_write=0
usb_read=0
usb_write=0
private_surface_export=0
hidden_surface_export=0
restricted_surface_export=0
secret_surface_export=0
repo_publish=0
package_release=0
```

Describe-only fields may be open where appropriate:

```text
tool_describe=1
skill_read=1
browser_observe=1
```

These are metadata descriptors, not runtime permissions.

## Verification

CI verifies every `packs/*/*.hbp` file with both:

```text
python -m hyperbehcs_hermes.cli verify <pack>
python -m hyperbehcs_hermes.cli verify-chain <pack>
```

This means descriptor expansion must remain packet-valid, sidecar-valid, chain-valid, and fail-closed.
